# HOW TO START A NEW CHAT — RUKIA FAST MODE

## Objetivo

Garantir continuidade arquitetural sem degradar performance do navegador ou perder sinergia técnica.

O histórico extenso pode tornar o chat lento.  
Este protocolo mantém velocidade e organização.

---

# REGRA PRINCIPAL

Um chat = Uma fase ou um documento.

Nunca misturar múltiplas fases em um único chat.

---

# COMO INICIAR UM NOVO CHAT

Sempre começar com o seguinte formato:

---

Título sugerido do chat:
RUKIA_FXX_NOME_DA_FASE

Exemplo:
- RUKIA_F03_ARQUITETURA
- RUKIA_F04_MODELO_DADOS
- RUKIA_F06_VERTICAL_SLICE

---

Mensagem inicial obrigatória:

1. Fase atual:
2. Último documento fechado:
3. Último commit realizado:
4. Objetivo do chat atual:
5. Link do repositório (opcional):

---

# EXEMPLO

Fase atual: F06 — Vertical Slice  
Último documento fechado: API_CONTRACTS_MVP_V1.md  
Último commit: docs: definir contratos de API do MVP v1  
Objetivo: Definir ordem de implementação do backend  

---

# REGRAS DE PERFORMANCE

1. Uma resposta = um bloco estruturado.
2. Evitar anexos grandes.
3. Evitar prints desnecessários.
4. Código somente quando necessário.
5. Documento no GitHub é a fonte da verdade.

---

# PRINCÍPIO ARQUITETURAL

A memória oficial do projeto não está no chat.  
Está no repositório versionado.

O chat é apenas ferramenta de execução.

---

# RESULTADO ESPERADO

- Chat leve
- Continuidade técnica
- Decisões rastreáveis
- Execução rápida
